# RiskOpsBench 1.0: Corpus Documentation

This document describes the structure of the RiskOpsBench 1.0 public sample corpus. The sample corpus is a 100-row subset of the cybersecurity domain corpus, generated by USGE v2.6 from the public sample Input Manifest.

---

## Corpus Composition

The 100-row sample corpus covers the Enterprise Cybersecurity Threat Operations domain. It contains 71 pure cybersecurity domain rows and 29 rows compiled from an expanded multi-domain manifest. The sample assessment report covers the 71 pure cybersecurity rows, which represent what a clean single-domain compilation produces.

## Difficulty Distribution

| Tier | Count | Interpretation |
|---|---|---|
| EASY | 5 | Structurally simple with counterintuitive correct outcomes. Below-chance accuracy on EASY is a strong negative signal. |
| MEDIUM | 32 | Near-chance performance is the expected failure mode. Fragile Consensus and Partial Observability challenge types dominate. |
| HARD | 13 | High structural complexity. Below-chance performance is common and reflects systematic misclassification. |
| EXPERT | 21 | Highest declared difficulty but cleaner signal structure beneath. Systems with strong evidence attribution resolve these more reliably than MEDIUM or HARD. |

Difficulty reflects structural complexity, not intuitive difficulty. The difficulty-accuracy relationship is frequently inverted: EXPERT accuracy exceeds EASY accuracy in systems without strong evidence attribution.

---

## Challenge Type Distribution

| Challenge Type | Count | Design Purpose |
|---|---|---|
| Deception | 13 | Surface signals support the incorrect classification. Correct outcome requires evidence attribution below the surface signal layer. |
| Evidence Conflict | 13 | Evidence objects point in contradictory directions. Correct classification requires weighted resolution by source reliability. |
| Fragile Consensus | 12 | Majority observer agreement supports the incorrect classification. Systems following consensus without independent evidence verification fail. |
| Partial Observability | 12 | Observer visibility gaps create structural blind spots. Reasoning about unobserved evidence is required. |
| Sensor Failure | 8 | Critical evidence is masked across specific observer nodes. Fallback reasoning architecture is required. |
| Uncertainty Spike | 8 | Posterior distributions are volatile. Confidence must be calibrated to signal uncertainty. |
| Standard Baseline | 5 | Structurally simple cases with clean signals. Below-chance accuracy here is the strongest negative signal in an assessment. |

## Benchmark Split Distribution

| Split | Count | Purpose |
|---|---|---|
| adversarial_test | 34 | Surface signals specifically constructed to support the incorrect classification. |
| long_tail_imbalance | 32 | ~87.5% legitimate, ~12.5% malicious. Accuracy alone is not a valid metric on this partition. |
| test | 5 | Standard partition for baseline comparison. |

## Observer Architecture

| Parameter | Value |
|---|---|
| Observers per scenario | 18 |
| Base trust per observer | 0.85 |
| Average hidden evidence per observer | 1.66 |
| Average visible evidence per observer | 3.32 |
| Partial observability design | No observer has full visibility of all evidence in any scenario |

Observer disagreement is structurally produced by the partial observability architecture. It is not introduced by observer error. Systems that follow majority observer consensus without independent evidence attribution will systematically fail adversarial and fragile consensus scenarios.

## Corpus Row Structure

Each corpus row contains the following fields:

| Field | Description |
|---|---|
| truth_label | Ground-truth classification for the scenario |
| difficulty | EASY, MEDIUM, HARD, or EXPERT |
| challenge_type | One of the seven challenge types above |
| benchmark_split | adversarial_test, long_tail_imbalance, or test |
| evidence_objects_json | Full evidence object definitions with type, strength, and hypothesis support direction |
| gold_critical_evidence | Ground-truth critical evidence objects for the scenario |
| criticality_distribution_map | Evidence criticality ranking |
| counterfactual_rankings_map | Evidence objects ranked by descending delta impact on the posterior |
| consensus_bayesian_trace | Complete reasoning chain from prior through likelihood ratio product to final posterior |
| consensus_confidence | Expressed confidence of the consensus classification |
| calibration_class | OVERCONFIDENT or WELL_CALIBRATED |
| quality_score | Structural integrity score for the scenario compilation |
